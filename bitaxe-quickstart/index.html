Bitaxe Quick Start

This is a no-fluff setup guide for new Bitaxe owners. The goal is simple: get out of the box and mining correctly, without guessing or tuning.

1. Power On & First Boot

Plug in the Bitaxe.

After about 30 seconds it creates a Wi-Fi network named something like Bitaxe-XXXX.

Connect to that network from your phone or computer. A setup page usually opens automatically.

If it doesn’t open, go to http://192.168.4.1.

2. Network Setup (One Time)

  • Hostname: use something simple like gamma01 or nq01
  • Your Wi-Fi SSID
  • Your Wi-Fi password

Save and reboot.

Hostnames help you identify devices later in your network and pool dashboards.

3. Accessing AxeOS

After reboot, the miner connects to your network via DHCP.

Find its IP address using the device screen or your router’s connected-devices list, then open:

http://x.x.x.x

If you see hashrate, temperatures, and pool status, you’re in the right place.

4. Pool Configuration (Required)

  • Stratum Host: e.g. solo.ckpool.org
  • Stratum Port: e.g. 3333
  • Stratum User:
    bc1qxxxxxxx.gamma01
  • Stratum Password: x (usually ignored)

The wallet address determines payouts. The suffix after the period names the device.

Fill in the Fallback pool fields as well to allow automatic failover.

Save and reboot. You are now mining.

5. What Not to Change (Yet)

  • Frequency
  • Voltage
  • Difficulty
  • ExtraNonce settings
  • Advanced logging

Each Bitaxe ships with conservative, known-good defaults. Leave them alone during first setup.

6. Burn-In Period

Let the miner run for a few hours before tuning anything.

  • Watch hashrate stability
  • Watch temperatures
  • Watch reject rate

This gives you baseline data. Without a baseline, tuning is guessing.

7. What’s Normal

  • Hashrate fluctuation
  • Occasional rejected shares
  • Random “best difficulty” jumps
  • Temperatures rising as the miner warms up

Stable operation over time is what matters.

Final Advice

If it’s hashing steadily and temperatures look reasonable, leave it alone.

Uptime comes first. Tuning comes later.
